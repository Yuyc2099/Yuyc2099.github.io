# 定点数与 Q 格式

## 1. 为什么使用定点数

定点数使用整数模拟小数，常用于没有浮点运算单元，或者对执行时间、存储空间有严格要求的 MCU。

Qn 表示数值中包含 n 个二进制小数位：

```text
存储整数 = 真实值 × 2^n
真实值   = 存储整数 ÷ 2^n
```

Qn 不是把一个单位分成 n 份，而是分成 `2^n` 份。

| 格式 | 缩放倍数 | 最小步长 |
| --- | ---: | ---: |
| Q12 | 4096 | 1 / 4096 |
| Q15 | 32768 | 1 / 32768 |
| Q16 | 65536 | 1 / 65536 |
| Q24 | 16777216 | 1 / 16777216 |

例如，真实值 `0.5` 的存储形式为：

```text
Q12(0.5) = 2048
Q15(0.5) = 16384
Q16(0.5) = 32768
Q24(0.5) = 8388608
```

## 2. 精度与范围

Q 格式只能表示离散数值，不能落在离散点上的真实数必须量化，因此会产生精度损失。

```text
Q15 最小步长 = 1 / 32768 ≈ 0.00003052
Q16 最小步长 = 1 / 65536 ≈ 0.00001526
```

量化方式不同，最大误差也不同：

```text
直接截断：误差小于 1 个最小步长
四舍五入：误差不超过 0.5 个最小步长
```

小数位越多：

- 小数精度越高；
- 使用相同整数类型时，可表示的整数范围越小；
- 乘法和左移越容易溢出。

需要区分“Q 格式”和“存储类型”。Q15 表示按 `2^15` 缩放，并不表示变量总共只有 15 位。Q15 可以存放在 `int16_t`、`int32_t` 或 `int64_t` 中，它们的缩放规则相同，但可表示范围不同。

如果使用有符号 W 位整数保存 Qn，理论范围约为：

```text
最小值 = -2^(W - n - 1)
最大值 =  2^(W - n - 1) - 2^(-n)
```

## 3. 定义和转换

可以定义常用缩放系数和转换宏：

```c
#include <stdint.h>

#define Q12_SCALE (1L << 12)
#define Q15_SCALE (1L << 15)
#define Q16_SCALE (1L << 16)
#define Q24_SCALE (1L << 24)

#define Q12(x) ((int32_t)((x) * Q12_SCALE))
#define Q15(x) ((int32_t)((x) * Q15_SCALE))
#define Q16(x) ((int32_t)((x) * Q16_SCALE))
#define Q24(x) ((int32_t)((x) * Q24_SCALE))
```

浮点数转换为 Q15：

```c
int32_t float_to_q15(float value)
{
    return (int32_t)(value * 32768.0f);
}
```

Q15 转换为浮点数：

```c
float q15_to_float(int32_t value)
{
    return (float)value / 32768.0f;
}
```

示例：

```c
int32_t fixed = float_to_q15(0.5f);  // 16384
float real = q15_to_float(fixed);    // 0.5
```

### 3.1 正数示例：表示 10.1

使用 Q15：

```text
Q15 缩放倍数 = 32768
10.1 × 32768 = 330956.8
```

C 语言直接强制转换会向 0 截断：

```c
int32_t value_q15 = (int32_t)(10.1f * 32768.0f);
// value_q15 = 330956
```

还原后的值为：

```text
330956 / 32768 = 10.0999755859375
```

如果使用四舍五入，存储整数为 `330957`：

```text
330957 / 32768 = 10.100006103515625
```

使用 Q16：

```text
Q16 缩放倍数 = 65536
10.1 × 65536 = 661913.6
```

截断结果：

```text
661913 / 65536 = 10.0999908447265625
```

四舍五入结果：

```text
661914 / 65536 = 10.100006103515625
```

Q 格式只能表示离散点，因此 `10.1` 通常不能被精确表示，只能保存为相邻的近似值。

### 3.2 负数示例：表示 -0.2

负数使用有符号整数保存，底层由补码表示。进行 Q 格式转换时不需要手动处理符号位。

使用 Q15：

```text
-0.2 × 32768 = -6553.6
```

C 语言强制转换向 0 截断：

```c
int32_t value_q15 = (int32_t)(-0.2f * 32768.0f);
// value_q15 = -6553
```

还原后的值为：

```text
-6553 / 32768 = -0.199981689453125
```

四舍五入到最近整数时为 `-6554`：

```text
-6554 / 32768 = -0.20001220703125
```

使用 Q16：

```text
-0.2 × 65536 = -13107.2
```

最接近的整数为 `-13107`：

```text
-13107 / 65536 = -0.1999969482421875
```

### 3.3 截断与四舍五入转换

截断转换：

```c
#include <stdint.h>

int32_t float_to_q15_trunc(float value)
{
    return (int32_t)(value * 32768.0f);
}

int32_t float_to_q16_trunc(float value)
{
    return (int32_t)(value * 65536.0f);
}
```

四舍五入转换：

```c
#include <stdint.h>
#include <math.h>

int32_t float_to_q15_round(float value)
{
    return (int32_t)lroundf(value * 32768.0f);
}

int32_t float_to_q16_round(float value)
{
    return (int32_t)lroundf(value * 65536.0f);
}
```

### 3.4 存储宽度决定可表示范围

能否表示 `10.1` 不仅取决于 Q15/Q16，还取决于底层整数类型的宽度。

传统的 `int16_t` Q15 使用 1 个符号位和 15 个小数位：

```text
范围约为 -1.0 ～ 0.999969482421875
```

因此不能表示 `10.1`：

```c
int16_t value = (int16_t)(10.1f * 32768.0f); // 超出范围
```

如果使用 `int32_t` 保存 Q15，范围约为：

```text
-65536.0 ～ 65535.999969482421875
```

因此可以表示 `10.1`。可以记住：

```text
Qn 决定小数精度
整数类型的位数决定整体范围
```

## 4. Q 格式对齐

不同 Q 格式不能直接进行加、减或比较，必须先对齐小数位。

### 4.1 Q15 转 Q16

Q16 比 Q15 多一个小数位，因此左移一位：

```c
int32_t q15_to_q16(int32_t value)
{
    return (int32_t)((int64_t)value * 2);
}
```

例如：

```text
Q15(0.5) = 16384
16384 << 1 = 32768 = Q16(0.5)
```

左移不会损失小数精度，但可能溢出。需要更安全的中间结果时，可以先提升到 `int64_t`：

```c
int64_t q16 = (int64_t)q15 * 2;
```

### 4.2 Q16 转 Q15

Q16 比 Q15 多一个小数位，因此右移一位：

```c
int32_t q16_to_q15_trunc(int32_t value)
{
    return value >> 1;
}
```

这会丢弃最低位，可能产生量化误差。

带正负数四舍五入的转换：

```c
int32_t q16_to_q15_round(int32_t value)
{
    if (value >= 0)
    {
        return (value + 1) >> 1;
    }

    return (int32_t)(-(((-(int64_t)value) + 1) >> 1));
}
```

## 5. 加法

### 5.1 相同 Q 格式相加

相同 Q 格式可以直接相加：

```c
int32_t q15_add(int32_t a, int32_t b)
{
    return a + b;
}
```

示例：

```c
int32_t a = Q15(0.3f);
int32_t b = Q15(0.2f);
int32_t c = a + b;       // Q15(0.5)
```

虽然格式不变，但加法结果仍可能超出存储类型范围。

### 5.2 Q15 与 Q16 相加

应先统一格式。通常优先把 Q15 提升为 Q16，以保留更多精度：

```c
int32_t add_q15_q16_to_q16(int32_t a_q15, int32_t b_q16)
{
    return (int32_t)((int64_t)a_q15 * 2 + b_q16);
}
```

计算 `0.5 + 0.5`：

```text
a_q15      = 16384
a_q15 << 1 = 32768 = Q16(0.5)
b_q16      = 32768 = Q16(0.5)
结果        = 65536 = Q16(1.0)
```

如果担心中间值溢出，可以使用 `int64_t`：

```c
int64_t result_q16 = (int64_t)a_q15 * 2 + b_q16;
```

## 6. 减法

减法规则与加法相同：参与运算的数必须使用相同 Q 格式。

相同 Q15 格式相减：

```c
int32_t q15_sub(int32_t a, int32_t b)
{
    return a - b;
}
```

Q15 减 Q16，并输出 Q16：

```c
int32_t sub_q15_q16_to_q16(int32_t a_q15, int32_t b_q16)
{
    return (int32_t)((int64_t)a_q15 * 2 - b_q16);
}
```

Q16 减 Q15，并输出 Q16：

```c
int32_t sub_q16_q15_to_q16(int32_t a_q16, int32_t b_q15)
{
    return (int32_t)((int64_t)a_q16 - (int64_t)b_q15 * 2);
}
```

## 7. 乘法

乘法后，小数位数相加：

```text
Qa × Qb = Q(a+b)
```

例如：

```text
Q15 × Q15 = Q30
Q15 × Q16 = Q31
Q12 × Q16 = Q28
```

### 7.1 Q15 乘 Q15，输出 Q15

原始乘积是 Q30。右移 15 位后得到 Q15：

```c
int32_t mul_q15(int32_t a_q15, int32_t b_q15)
{
    return (int32_t)(((int64_t)a_q15 * b_q15) >> 15);
}
```

计算 `0.5 × 0.5`：

```text
a = 16384 = Q15(0.5)
b = 16384 = Q15(0.5)

a × b       = 268435456，格式为 Q30
(a × b)>>15 = 8192，格式为 Q15
8192 / 32768 = 0.25
```

### 7.2 Q15 乘 Q16

原始乘积是 Q31。

输出 Q15，需要右移 16 位：

```c
int32_t mul_q15_q16_to_q15(int32_t a_q15, int32_t b_q16)
{
    return (int32_t)(((int64_t)a_q15 * b_q16) >> 16);
}
```

输出 Q16，需要右移 15 位：

```c
int32_t mul_q15_q16_to_q16(int32_t a_q15, int32_t b_q16)
{
    return (int32_t)(((int64_t)a_q15 * b_q16) >> 15);
}
```

### 7.3 通用乘法规则

两个数分别为 Qa 和 Qb，希望输出 Qc：

```text
右移位数 = a + b - c
```

当右移位数大于等于 0：

```c
result_qc = ((int64_t)a_qa * b_qb) >> (a + b - c);
```

如果 `a + b - c` 小于 0，则需要左移，但必须特别检查溢出。

## 8. 除法

整数除法会丢失小数，因此通常应在除法前左移，预留目标格式所需的小数位。

### 8.1 Q15 除以 Q15，输出 Q15

```c
int32_t div_q15(int32_t a_q15, int32_t b_q15)
{
    if (b_q15 == 0)
    {
        return 0;  // 实际项目应按需求处理除零错误
    }

    return (int32_t)(((int64_t)a_q15 * (1LL << 15)) / b_q15);
}
```

计算 `0.5 ÷ 0.25`：

```text
a = 16384 = Q15(0.5)
b = 8192  = Q15(0.25)

(a << 15) / b = 65536
65536 / 32768 = 2.0
```

### 8.2 Q15 除以 Q16

输出 Q15：

```c
int32_t div_q15_q16_to_q15(int32_t a_q15, int32_t b_q16)
{
    if (b_q16 == 0)
    {
        return 0;
    }

    return (int32_t)(((int64_t)a_q15 * (1LL << 16)) / b_q16);
}
```

输出 Q16：

```c
int32_t div_q15_q16_to_q16(int32_t a_q15, int32_t b_q16)
{
    if (b_q16 == 0)
    {
        return 0;
    }

    return (int32_t)(((int64_t)a_q15 * (1LL << 17)) / b_q16);
}
```

### 8.3 通用除法规则

被除数为 Qa，除数为 Qb，希望输出 Qc：

```text
左移位数 = b + c - a
```

当左移位数大于等于 0：

```c
result_qc = ((int64_t)a_qa * (1LL << (b + c - a))) / b_qb;
```

必须检查：

- 除数是否为 0；
- 左移后的中间值是否溢出；
- 结果是否超出目标整数类型范围；
- 是否需要四舍五入或饱和处理。

## 9. 比较运算

相同 Q 格式可以直接比较：

```c
if (a_q15 > b_q15)
{
}
```

Q15 与 Q16 比较前应先对齐：

```c
if ((int64_t)a_q15 * 2 > b_q16)
{
}
```

使用 `int64_t` 可以避免格式对齐时在 `int32_t` 范围内溢出。

## 10. 一阶低通滤波器示例

一阶低通滤波器公式：

```text
y[k] = α × x[k] + (1 - α) × y[k-1]
```

其中：

- `x[k]`：当前输入；
- `y[k-1]`：上一次输出；
- `y[k]`：本次输出；
- `α`：0～1 之间的滤波系数。

当 `α` 较小时，输出变化慢、滤波更强；当 `α` 较大时，输出响应快、滤波更弱。

使用 Q16 表示 `α`：

```text
Q16(1.0)  = 65536
Q16(0.5)  = 32768
Q16(0.25) = 16384
Q16(0.1)  ≈ 6554
```

定点实现：

```c
#include <stdint.h>

#define Q_FORMAT 16
#define Q_SCALE  (1L << Q_FORMAT)

int32_t low_pass_filter(int32_t input,
                        int32_t previous_output,
                        int32_t alpha_q16)
{
    int32_t input_part = (int32_t)(((int64_t)alpha_q16 * input) >> Q_FORMAT);
    int32_t history_part = (int32_t)(((int64_t)(Q_SCALE - alpha_q16) * previous_output) >> Q_FORMAT);

    return input_part + history_part;
}
```

如果输入和历史输出是 Q12：

```text
Q16 × Q12 = Q28
Q28 >> 16 = Q12
```

所以函数返回值仍为 Q12。Q16 只用于表示滤波系数，不会改变输入和输出的最终 Q 格式。

## 11. 舍入

直接右移属于截断。对于正数，可以在右移前加半个最低保留单位实现四舍五入。

例如，把 Q30 乘积恢复为 Q15：

```c
int32_t mul_q15_round(int32_t a_q15, int32_t b_q15)
{
    int64_t product_q30 = (int64_t)a_q15 * b_q15;
    product_q30 += (1LL << 14);
    return (int32_t)(product_q30 >> 15);
}
```

上述简单写法主要适用于非负数。需要同时正确处理正负数时，应根据符号分别添加或减去舍入偏置。

## 12. 饱和处理

普通整数溢出可能导致结果回绕。控制系统通常希望结果停留在允许的最大值或最小值。

将 64 位中间结果饱和到 32 位：

```c
#include <stdint.h>
#include <limits.h>

int32_t saturate_i64_to_i32(int64_t value)
{
    if (value > INT32_MAX)
    {
        return INT32_MAX;
    }

    if (value < INT32_MIN)
    {
        return INT32_MIN;
    }

    return (int32_t)value;
}
```

## 13. 常见风险

### 13.1 中间乘积溢出

不推荐：

```c
int32_t result = (a * b) >> 15;
```

推荐：

```c
int32_t result = (int32_t)(((int64_t)a * b) >> 15);
```

### 13.2 不同格式直接相加

错误：

```c
int32_t result = value_q15 + value_q16;
```

正确：

```c
int64_t result_q16 = (int64_t)value_q15 * 2 + value_q16;
```

### 13.3 除零

所有定点除法都必须先检查除数是否为 0。

### 13.4 有符号右移

负有符号整数右移在不同语言标准和实现中可能存在差异。嵌入式项目应确认编译器行为，或者使用明确的辅助函数处理负数。

### 13.5 左移负数或溢出

对负有符号数左移，以及左移后超出有符号整数范围，在 C 语言中可能产生未定义行为。更稳妥的方法是先提升到更宽的整数类型，并检查范围。

## 14. 运算规则速查

```text
真实值转 Qn：真实值 × 2^n
Qn 转真实值：存储整数 ÷ 2^n

加法：先统一 Q 格式，再直接相加
减法：先统一 Q 格式，再直接相减
乘法：Qa × Qb = Q(a+b)
除法：根据目标格式先左移，再执行整数除法

Q15 → Q16：左移 1 位
Q16 → Q15：右移 1 位，会损失最低位

Q15 × Q15 → Q15：乘积右移 15 位
Q15 × Q16 → Q15：乘积右移 16 位
Q15 × Q16 → Q16：乘积右移 15 位

乘法中间值：优先使用 int64_t
除法：必须检查除零和左移溢出
输出：根据需求进行舍入、限幅或饱和
```
