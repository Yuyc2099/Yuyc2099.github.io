# 文章标题编号

## 编号规则

- 将同目录 `post.json` 中的 `title` 作为文章标题，并让 Markdown 首行写成相同的 `# 一级标题`，不添加章节编号。
- 将 Markdown `##` 视为一级章节，直接写成 `## 1. 标题`、`## 2. 标题`。
- 将 Markdown `###` 视为二级章节，直接写成 `### 1.1 标题`、`### 1.2 标题`。
- 不跳过标题层级；每遇到新的 `##`，从 `.1` 重新计算其下的 `###`。
- 以 Markdown 中的编号为准，让正文标题、锚点和“本文目录”自然保持一致。

## 编辑要求

1. 新增或整理文章时，正文与元数据分离：`.md` 只保存带一级标题的正文，博客元数据保存到同目录的 `post.json`，不使用 Front Matter。
2. 使用 `scripts/number-headings.mjs` 统一移除旧数字前缀并重新编号，避免重复编号或插入章节后的跳号。
3. 不把围栏代码块中以 `#` 开头的注释或命令识别为文章标题。
4. 遇到 `####` 时先确认它确实是三级内容层级；除非用户要求，不修改其编号。
5. 修改标题导致锚点变化时，搜索并同步更新指向旧锚点的文内链接。
6. 文章不需要封面或占位 SVG；只有正文实际引用图片时才创建 `images/`。

## 检查流程

1. 对单篇文章运行 `node .agents/skills/number-article-headings/scripts/number-headings.mjs <文章路径>`；整理全部文章时传入 `content/posts`。
2. 运行同一命令并加上 `--check`，确认不再需要调整。
3. 运行 `npm run build`。
4. 检查 Markdown 首行的 `# 一级标题` 是否与 `post.json.title` 一致。
5. 检查生成页面中的 `h2` 是否以 `N.` 开头，`h3` 是否以 `N.N` 开头。
6. 检查“本文目录”与正文一致，并抽查代码块内容未被修改。
